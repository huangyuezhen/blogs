掌贝运维开发测试环境说明

一些约定

对于测试环境还有生产环境，我们按以下约定的目录去进行文件存放

如无特殊，我们的几大系统，都是按以下目录去存放对应的文件
/data/logs/   项目的日志文件都在此目录
/data/conf   项目的配置文件都在此处
/data/svr  项目源码存放位置
/data/sh   项目的脚本存放位置

前言：

  • 运维开发主要有三种运行，开发环境，测试环境，生产环境，都有对应的文档进行了介绍，下面主要是补充一下测试环境的账号信息等内容

10.10.50.30 root / pUaj3TBJm3sPhceb

三大系统使用的测试环境数据库都是10.10.50.30 上的mysql 实例，账号 remote_test /remote_passwd ，mysql root 账号在10.10.50.30主机上可以直连，没设置密码。

publish

条件

publish-server 的部署主机需要与测试主机（我们目前用的是 10.10.51.41-44 ）之间需要是可以互相访问的，需同时满足两个条件：

  • 部署主机的product 用户能够直接ssh 到测试主机的 product 用户下（即建立ssh 互信）

  • 测试主机 能够通过 部署主机的IP 进行拉包等操作

其他

在测试主机10.10.50.30 测试主机上，已建好的 python虚拟环境是 /data/svr/python36publishEncoding/bin/python

可直接使用来运行publish-server

  • 测试环境部署信息，参见wiki:

http://wiki.corp.yunnex.com/pages/viewpage.action?pageId=21304910

  • 测试环境访问地址 http://10.10.50.30:8070

  • 测试环境的publish 需要依赖于测试环境的cmdb（ http://10.10.50.30/）

  • 如果需要在测试环境下发版，需要在测试cmdb 上把对应的应用绑定到四台测试主机（10.1051.41-44）上，即可发版。

  • 发布系统对不符合版本规范（如：待发版本号低于回滚版本等），不强制要求，只弹框提醒发版者注意

  • 测试环境flower 访问地址： http://10.10.50.30:5555

flower 是 celery 运行 tasks的一个监控工具，一个命令即可运行起来，无需嵌入代码

详见https://flower-docs-cn.readthedocs.io/zh/latest/index.html

生产由于一般通过log 形式即可判断 问题，所以flower 没安装在生产环境，主要安装在测试环境做tasks 调试监控使用

  1. 安装 node-sass 报错

出现的问题：

`` npm ERR! node-sass@4.9.3 postinstall:node scripts/build.js` npm ERR! Exit status 1 npm ERR! npm ERR! Failed at the node-sass@4.9.3 postinstall script. npm ERR! This is probably not a problem with npm. There is likely additional logging output above.

npm ERR! A complete log of this run can be found in: npm ERR! C:\Users\87217\AppData\Roaming\npm-cache_logs\2020-04-22T10_04_59_535Z-debug.log ```

解决方法：

解决方法： 1、卸载node-sass 运行：npm uninstall node-sass 2、重新安装指定的node-sass版本(安装失败的版本) 运行：npm install node-sass@4.9.3 3、如果还是安装不成功的话，或者安装太慢连接不上，使用一下淘宝镜像后重复1,2步骤，成功后再npm install 使用淘宝镜像运行：npm config set registry https://registry.npm.taobao.org
