vue 项目文件结构介绍

$ npm install -g vue-cli
$ vue init webpack my-project
$ cd my-project
$ npm install
$ npm run dev

一个vue 项目是有固定文件结构的，使用 vue init webpack my-project 命令可以初始化一个具备vue 完整架构的文件目录，然后我们到相应的目录下去添加修改文件即可。

下面讲讲 各个

  • build

    webpack 打包静态文件的配置，包含dev /test/product 三个环境的配置

  • config - webpack config files

    webpack 打包静态文件的配置，包含dev /test/product 三个环境的配置

  • dist

    这个文件夹在初始化的时候是没有的，只有运行了npm run build 命令后，才会出现这个目录。打包好的静态文件就存放在此处，部署到服务器上时，就是采用这里的文件。

  • src #src 才是我们修改前端vue 源码的真正地方，下面设很多个目录

    • api #存放前端调用api 的地方
    • assets # 一些资源文件 如 图片等
    • common # 一些公用方法
    • components - # 组件，vue 的一个特殊之处就是可以使用组件，组件的作用类似于html的函数，作用是复用
    • styles # css 文件
    • views # 定义 vue 页面的地方，主力开发就在此处
    • store # 请求后端返回的数据，在此处全局存放
    • App.vue # 入口页面
    • main.js #入口 加载组件 初始化等
    • routes.js # 路由定义

  • static # 一些静态文件存放地方，我们几个项目此目录都是空的

  • index.html # 整个vue 唯一的一个html,把 Vue 应用通过 main.js挂载到一个 DOM 元素上

index.html 文件内容

<!DOCTYPE html>
<html>
  <head>
    <meta charset="utf-8">
    <link rel="shortcut icon" type="image/png" href="/static/favicon.png" />
    <meta name="viewport" content="width=device-width,initial-scale=1.0">
    <title>title text</title>
  </head>
  <body style="margin: 0px">
    <div id="app"></div>
    <!-- built files will be auto injected -->
  </body>
</html>

main.js 内容

/* main.js */

import App from './App' 
/* 把App.vue 页面import 进来 /*

new Vue({
  el: '#app',    /* 此处 上文的 index.html 进行挂载，通过 app这个id/* 
  render: h => h(App)   /* 把Vue 应用会将其挂载到一个 DOM 元素上
})

Vue Router

vue 的设计思想跟server 端的很类似，它的router 跟 后端的也是同样的含义，根据请求的uri 路由到不同的页面去，基本上也可以认为vue router 是我们前端文件的大门。

那么 vue router 是怎么引用起来的呢？也是类似，通过main.js 文件中

import router from './router'
new Vue({
  el: '#app',
  router,          /* 此处把 router绑定到 vue 应用上
  render: h => h(App)
})

router/index.js 

import Vue from 'vue'
import Router from 'vue-router'
import Main from '@/views/main'
import CreatePublishPlan from '@/views/pages/CreatePublishPlan'

Vue.use(Router)

export const navRoutes = [
  {
    path: 'create-plan',
    component: CreatePublishPlan, // uri: create-plan  与我们的CreatePublishPlan 组件绑定
    title: '创建发版',
    name: 'CreatePlan',
    icon: 'el-icon-edit-outline',
    meta: { cache: true }
  }
]

export default new Router({
  mode: 'history', // require service support
  routes: [
    {
      path: '/',
      name: 'Home',
      component: Main, // 菜单入口
      children: navRoutes  // 嵌套路由
    }
  ]
})

使用router-view 使得菜单栏一直保持在页面上，请注意看 文中的注释
